Git命令

发表于 2021-02-17 分类于 计算机 ， 其他
Git命令简介

Git

基本操作指令：add - commit - pull - push要按这个顺序敲

基本概念

  • 工作目录Working Directory：被Git管理的项目目录
  • 暂存区Staging Area：临时存放被修改文件
  • Git仓库Repository：存放提交记录

使用前配置

修改配置只需重复以下命令即可,配置后可到系统文件目录C:\Users\用户名下的.gitconfig文件查看配置信息

  • 配置提交人姓名 git config --global user.name 昵称
  • 配置提交人邮箱 git config --global user.email 邮箱
  • 查看配置信息 git config --list
  • 终端中文支持 git config --global core.quotepath false

提交

  • 修改本地仓库默认分支名 git config --global init.defaultBranch main

  • 初始化Git仓库 git init

  • 查看文件状态 git status

  • 添加文件到暂存区(追踪文件) git add 文件名.后缀名

    git add . 会把当前工作目录下的所有文件添加到暂存区

    .gitignore文件夹存放想要被git add .命令忽略的文件或文件夹

  • 移交代码 git commit -m 提交说明信息

  • 查看移交记录 git log

撤销

  • 用暂存区中的文件覆盖工作目录中的文件,但保留暂存区 git checkout 文件名.后缀名
  • 将文件从暂存区中删除 git rm --cached 文件名.后缀名
  • 回退到某一次提交,暂存区和工作目录都回退到该次提交时的状态 git reset --hard 移交ID
    • 移交ID 是通过git log得到的日志中的commit唯一编码值

分支

分支分类

  • 主分支main：第一次向仓库移交时自动产生的分支
  • 开发分支develop：基于marster分支创建,作为开发的分支
  • 功能分支feature：基于develop分支创建,作为具体功能的分支

分支命令

新分支要在远程仓库可见需要用git push(见后文)

  • 查看分支 git branch
  • 创建分支 git branch 分支名称
  • 切换分支 git checkout 分支名称 (切换分支前应当先移交commit,否则会导致分支混淆)
  • 合并分支 git merge 旁支名称 (合并分支后原分支仍存在)
  • 删除分支 git branch -d 分支名称 (分支被合并后才能删除,强制删除可用git branch -D 分支名称)

暂时保存修改

可以暂时存储当前分支,临时转向其他分支,而不需要先commit当前分支

  • 临时存储当前分支 git stash (临时存储后即可使用 git checkout 分支名称 转向其他分支)
  • 在被存储分支下恢复之前的分支 git stash pop

Github

push

推送到远端仓库

首次push需要输入Github账户和密码,windows系统下会把密码存储在控制面板-凭据管理器-Windows凭据中

  • git push 远程仓库地址/地址别名 分支名称

  • git push -u 远程仓库地址/地址别名 分支名称 (-u 的作用是记住push的地址及分支,下次push只需输入命令git push即可)

  • git remote add 远程仓库地址别名 远程仓库地址

    链接远程仓库并设置别名

    多人协作流程

仓库拥有者需要在Github中设置其他用户的pull权限,步骤为Setting-Collaborations-输入对方账号-Add collaborator,之后把Copy invite lnnk发送给对方,对方用户打开后只需Accept invitation

clone

克隆远程仓库到本地,克隆的远程仓库也可以使用仓库拥有者设定的地址别名

  • git clone 远程仓库地址

pull

拉取远程仓库中最新的版本,pull作为读操作不需要验证用户身份,如果远程仓库版本高于本地仓库,则需要先pull后push,否则会出现冲突

  • git pull 远程仓库地址 分支名称

pull request

未经过仓库拥有者赋予pull权限的用户只能通过请求Pull requests对仓库进行修改(需要先fork到自己的仓库),经过仓库拥有者审核后由拥有者对该请求进行合并,这些操作都是在Github中进行

一方流程为：Pull requests - New pull request - Create pull request

一方流程为：Merge pull request

解决冲突

拉取远程仓库到本地后可能会出现冲突,其标识如下,冲突需要人工解决

SSH协议

密钥存储目录为C:\Users\用户名\.ssh

设置之后push的远程仓库地址可以换成SSH

  • 生成密钥 ssh-keygen

  • 公钥 id_rsa.pub 放置于Github中

    Setting - SSH and GPG keys - New SSH key - 粘贴公钥

  • 私钥 id_rsa
